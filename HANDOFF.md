# Grimoire — Handoff Brief

## What This Is
A two-part tool for navigating large codebases generated by agentic coding tools. It maps project directories into a searchable, annotated visual explorer with VS Code integration.

## Files
- **`grimoire.py`** — Python CLI that walks a real project directory, applies heuristic descriptions, calls the Claude API for AI-powered descriptions, and outputs `.grimoire.json`
- **`grimoire.jsx`** — React artifact (Claude.ai) that renders the interactive treemap UI. Accepts pasted file trees OR imported `.grimoire.json` files.
- **`vscode-extension/`** — VS Code extension with sidebar tree view, interactive webview map, search-by-tag, and click-to-open. See `vscode-extension/INSTALL.md`.

## Current State
- CLI tool: Feature-complete, tested logic, ready to use
- React UI: Feature-complete with treemap layout, search, breadcrumb nav, VS Code URI links, inline editing, export/import, floating toolbar (text size S/M/L, plain English toggle), and a clipboard-based AI description workflow
- The in-app API calls don't work (Claude artifact sandbox can't reach api.anthropic.com — CORS + network restrictions). This is why the CLI exists.

## Architecture Decisions
- CLI uses the `anthropic` Python SDK if available, falls back to raw `urllib` HTTP calls
- Descriptions are batched (default 30 items per API call) to stay within token limits
- Heuristics cover ~100+ common file/directory patterns as a fallback
- VS Code links use the `vscode://file/` URI scheme — no extension required
- `.grimoire.json` is the interchange format between CLI and UI
- All VS Code commands use the `grim.` prefix (e.g., `grim.scan`, `grim.annotateFile`)

## Tech Stack
- Python 3.10+ (CLI)
- React + Tailwind concepts via inline styles (UI)
- Anthropic Claude API (claude-sonnet-4-20250514)
- No build tools needed — CLI is a single file, UI is a single JSX artifact

## Key Commands
```bash
# Install SDK (optional but recommended)
pip install anthropic

# Run with API key
export ANTHROPIC_API_KEY="sk-ant-..."
python grimoire.py /path/to/project

# Heuristics only (no API calls)
python grimoire.py /path/to/project --no-ai

# Custom batch size and exclusions
python grimoire.py /path/to/project --batch-size 40 --exclude "logs,data"

# Annotate a file with inline comments
python grimoire.py annotate ./src/app.js --mode tutor
python grimoire.py annotate ./src/app.js --mode non-technical --in-place
python grimoire.py annotate ./src/utils.py --mode minimal --output ./docs/utils-commented.py
```

## Design Philosophy
This tool exists because agentic coding (Cursor, Claude Code, Copilot Workspace, etc.) generates massive codebases extremely fast, and developers — especially those newer to large project structures — need a "map" to navigate what was built. Think of it as Google Maps for your repo: you can zoom in from directory level to file level, search for functionality by description, and jump straight to the code.
